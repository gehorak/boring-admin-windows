# OPERATING-MANUAL.md

Operating Manual  
Windows Workstation — No Domain Environment

---

## Purpose

This document explains **how this computer is intended to be used
during normal daily operation**.

It is written for:

- end users
- system owners
- technicians who did not build the system

This manual does **not** explain technical implementation details.
It defines **rules of use**.

If these rules are followed, the system remains:

- stable
- secure
- easy to recover

---

## Account usage rules (important)

### Account types

This system uses two types of accounts:

- **Personal user account** (Microsoft account)
- **Administrator accounts** (restricted, maintenance-only)

---

### What you must do

- Use **only your personal user account** for daily work
- Sign in using your **Microsoft account**
- Store your files in locations synchronized by OneDrive

---

### What you must not do

- Do not use administrator accounts for daily work
- Do not install software using administrator privileges
  unless explicitly instructed
- Do not share your account password

> Administrator accounts exist only for system maintenance
> and recovery.

---

## Data handling rules

### Where to store files

The following locations are automatically synchronized:

- Documents
- Desktop
- Pictures

Files stored there are protected outside the device.

---

### Why this matters

If the computer:

- fails
- is lost or stolen
- must be reinstalled

your data can be restored automatically.

---

### What not to do

- Do not store important files only on the local disk
- Do not disable OneDrive synchronization

---

## Software installation rules

### Allowed

- Software installed by the administrator
- Approved tools required for your work

---

### Not allowed

- Installing random software from the internet
- “Free utilities” promising optimization or cleaning
- Replacing built-in security software
- System cleaners or registry tools

If you need additional software:

> **Contact the administrator.**

---

## Updates and restarts

### Automatic updates

- Windows updates are enabled by design
- Some updates require a restart

---

### Your responsibility

- Restart the computer when prompted
- Do not postpone restarts indefinitely

Updates keep the system:

- secure
- compatible
- supportable

---

## Temporary or guest access

In some situations, a **temporary access account** may be created.

Rules:

- Temporary accounts are short-lived
- Data created under these accounts is removed when the account is deleted
- Temporary accounts are not intended for long-term use

---

## If something goes wrong

### Expected incidents

The following events are considered normal operational risks:

- hardware failure
- software corruption
- malware or ransomware
- device loss or theft

---

### Standard response

1. The system is wiped
2. Windows is reinstalled
3. You sign in again
4. Your data is restored automatically

> Repair is optional.  
> Recovery is guaranteed.

---

## What you do not need to worry about

- Normal use will not damage the system
- You do not need to clean or optimize Windows
- You do not need additional security software

If something feels wrong or unexpected:

> **Stop and contact the administrator.**

---

## Golden rules (summary)

- Use only your personal account
- Keep your data in OneDrive
- Allow updates and restarts
- Do not install unapproved software
- Ask before changing system settings

---

## Final note

This computer is designed to be:

- boring
- predictable
- easy to recover

If this manual is followed,
**serious problems are unlikely**.
